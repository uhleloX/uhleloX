Normalizes filters set up before WordPress has initialized to X_Hook objects.

public static build_preinitialized_hooks(array<string|int, mixed> $filters) : array<string|int, X_Hook>
Description

The $filters parameter should be an array keyed by hook name, with values containing either:

  • A X_Hook instance
  • An array of callbacks keyed by their priorities

Examples:

$filters = array(
    'wp_fatal_error_handler_enabled' => array(
        10 => array(
            array(
                'accepted_args' => 0,
                'function'      => function() {
                    return false;
                },
            ),
        ),
    ),
);

x_filter_build_unique_id()

class-x-hook.php:588

Builds Unique ID for storage and retrieval.

private x_filter_build_unique_id(string $hook_name, callable|string|array<string|int, mixed> $callback, int $priority) : string
Description

The old way to serialize the callback caused issues and this function is the solution. It works by checking for objects and creating a new property in the class to keep track of the object and new objects of the same class that need to be added.

It also allows for the removal of actions and filters for objects after they change class properties. It is possible to include the property $wp_filter_id in your class and set it to "null" or a number to bypass the workaround. However this will prevent you from adding new classes and any new classes will overwrite the previous hook by the same class.

Functions and static method callbacks are just returned as strings and shouldn't have any speed penalty.

Parameters
$hook_name
(string) Unused. The name of the filter to build ID for.
$callback
(callable |string |array<string|int, mixed>) The callback to generate ID for. The callback may or may not exist.
$priority
(int) Unused. The order in which the functions associated with a particular action are executed.
